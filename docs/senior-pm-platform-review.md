# Senior PM Platform Review

## Executive View

FFR Property OS has the right foundations: property records, tenants, maintenance, compliance, utilities, finance, email, intake, calendar, agents, approvals, ledger and Business Memory. The next product leap is not another large module. It is a tighter daily operating loop:

1. Start the day in one command center.
2. See the brief, risks, decisions and lane health.
3. Launch the right agent in dry-run mode.
4. Approve only actions that create external, legal, financial or access commitments.
5. Record outcomes into the ledger and Business Memory.

That loop is now the primary product strategy.

## What To Simplify

- Keep Today as the default first screen for non-technical users.
- Move secondary tools behind operating lanes and next actions.
- Avoid making the team choose between many dashboards before they know what needs attention.
- Turn raw emails, WhatsApps and calendar entries into tasks, approvals, events and memory.
- Keep generated memory inspectable as normal files rather than hiding the business brain inside prompts.

## What To Add Next

### 1. Leasing Pipeline

Create first-class records for lead, source, property, unit, viewing, offer, contract, deposit and status. This will let the Leasing & Revenue Agent manage follow-up, pricing, conversion and 52 Old Elvet availability without relying on spreadsheets and WhatsApp.

### 2. Short-Let Control Room

Create a lane for 52 Old Elvet, 2 St Margarets, St Andrews, 7 Cathedrals and any short-let units. It should connect calendar, OTA listing state, cleaner/linen rota, access codes, guest messages, revenue targets and owner blocks.

### 3. Contractor Work Ledger

Create structured job sessions: contractor, workers, date, hours, property, scope, before/after photos, invoice and approval. This directly solves the Florin/day-rate issue and makes high-cost work auditable.

### 4. Property Data Completeness

Capture exact bedrooms, bathrooms, unit names, sleeps, parking, meter locations, access notes, utility suppliers, compliance requirements and listing links. Agents need this before they can safely answer or act.

### 5. Autonomy Promotion

Each agent workflow should move through stages: observe, classify, draft, recommend, request approval, execute, record. Do not switch a workflow to execution until it has repeated clean dry-run outcomes with approvals logged.

## Built In This Pass

- Today command brief with headline, summary and operating mode.
- Next-best actions ranked from live counts.
- One-click suggested agent runs from Today.
- Decision queue surfaced on the first screen.
- Operating lane health across inbox/admin, maintenance, compliance, lettings, finance/suppliers and memory/calendar.
- Autonomy score and gaps.
- OS Overview senior review with simplifications, operating principles, roadmap and constraints.

## Operating Principles

- Every action has an owner, source, deadline and approval state.
- Agents draft and monitor by default; humans approve commitments.
- The database and ledger are the source of record.
- Curated Business Memory is the shared operating brain.
- The team sees fewer screens while agents see more context.
