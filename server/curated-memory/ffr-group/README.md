# FFR Group Business Canon

This is the curated, source-controlled operating memory for FFR Group, PSB Properties Partnership, PSB52 Limited and 52 Old Elvet.

It follows the practical Karpathy-style "LLM wiki" pattern: keep business knowledge in normal files that humans and agents can read, diff, link, correct and refresh. Generated Business Memory snapshots copy this folder into `wiki/curated/ffr-group/` so Codex agents can read it beside live database, inbox, WhatsApp, calendar and ledger memory.

## Agent Start Here

- Treat "52 Old Albert" as an alias or typo for `52 Old Elvet` unless a human confirms otherwise.
- Read `entities-and-governance.md` before finance, ownership, tax, lender or board questions.
- Read `portfolio-and-properties.md` and `52-old-elvet.md` before property, tenancy, listing, maintenance or valuation questions.
- Read `people-contractors-and-culture.md` before drafting for, assigning to or escalating with the team.
- Read `lettings-marketing-and-viewings.md` before replying to leads, pricing lets or preparing viewing follow-ups.
- Read `systems-comms-and-ai.md` before email, WhatsApp, Codex, Gmail, calendar or automation work.
- Read `finance-utilities-and-risk.md` before supplier, arrears, cashflow, utilities, mortgage, debt or risk work.
- Read `agent-operating-rules.md` before any autonomous action.

## Source Discipline

This canon is curated from local files that were available on the machine, including WhatsApp exports, ChatGPT export text files, Codex session logs, project folders, operating manuals, valuation documents and finance model outputs. It is intentionally not a raw dump.

Do not store passwords, one-time codes, full private tenant/prospect contact details, or personal identity data here. Put canonical live records in the platform database, connected inboxes, uploaded documents and `business_event_ledger`.

When there is conflict:

1. Live source systems win for current facts: SQLite, connected Gmail/IMAP, Google Calendar, uploaded documents and bank/accounting exports.
2. This curated canon wins for operating principles, historical context and team-specific judgement unless newer evidence contradicts it.
3. Agents should cite the file path and, where possible, source note before making a recommendation.
